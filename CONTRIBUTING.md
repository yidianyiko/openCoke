# Contributing to openCoke

Thanks for your interest in contributing 🎉

## How to contribute

1. Fork the repository
2. Create a feature branch (`feat/your-change`)
3. Commit with clear messages
4. Open a Pull Request with context and test notes

## Development guidelines

- Never commit secrets (API keys, tokens, passwords, private keys)
- Keep runtime artifacts out of git (logs, db files, local env files)
- Prefer small, focused PRs
- Add/adjust tests for behavior changes

## Commit style

Use simple conventional style when possible:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `chore:` maintenance
- `refactor:` code cleanup without behavior change

## Security reporting

If you find a vulnerability, please avoid public disclosure first.
Open a private report using GitHub Security Advisory or contact the maintainer.
