# 提醒系统快速开始指南

## 概述

本指南帮助你快速了解和使用改进后的提醒系统.

## 主要改进

### 1. 统一时间格式 + 智能推理

**问题：** LLM 有时输出不规范的时间格式

**解决：** 
- 在 Prompt 中注入当前时间
- LLM 根据当前时间推理任意时间
- 统一使用 "YYYY年MM月DD日HH时MM分" 格式

**示例：**
```
当前时间：2025年12月23日15时30分

用户说："下午3点提醒我"
LLM 推理：已过下午3点 → 输出 "2025年12月24日15时00分"

用户说："明天早上9点提醒我"
LLM 推理：明天是12月24日 → 输出 "2025年12月24日09时00分"
```

### 2. 时间段提醒功能

**新功能：** 支持"从X点到Y点每隔Z分钟提醒"

**使用场景：**
- "工作时间每30分钟提醒我喝水"
- "从早上9点到下午5点，每小时提醒我休息"
- "今天下午每半小时提醒我"

## 用户使用示例

### 基础提醒

```
用户："明天早上9点提醒我开会"
系统：创建单次提醒，2025年12月24日09时00分

用户："每天早上8点提醒我吃药"
系统：创建每日提醒，每天08时00分

用户："30分钟后提醒我"
系统：创建单次提醒，30分钟后
```

### 时间段提醒

```
用户："工作时间每30分钟提醒我喝水"
系统：创建时间段提醒
- 时间段：周一到周五 09:00-18:00
- 间隔：每30分钟
- 触发：09:00, 09:30, 10:00, ..., 17:30, 18:00

用户："从早上9点到下午5点，每小时提醒我休息"
系统：创建时间段提醒
- 时间段：每天 09:00-17:00
- 间隔：每60分钟
- 触发：09:00, 10:00, 11:00, ..., 17:00
```

## 开发者使用

### LLM 调用 reminder_tool

#### 基础提醒

```python
# 单次提醒
reminder_tool(
    action="create",
    title="开会",
    trigger_time="2025年12月24日09时00分"
)

# 每日提醒
reminder_tool(
    action="create",
    title="吃药",
    trigger_time="2025年12月24日08时00分",
    recurrence_type="daily"
)
```

#### 时间段提醒

```python
# 工作时间每30分钟提醒
reminder_tool(
    action="create",
    title="喝水",
    trigger_time="2025年12月24日09时00分",
    recurrence_type="interval",
    recurrence_interval=30,
    period_start="09:00",
    period_end="18:00",
    period_days="1,2,3,4,5"  # 工作日
)

# 每天上午每小时提醒
reminder_tool(
    action="create",
    title="休息",
    trigger_time="2025年12月24日09时00分",
    recurrence_type="interval",
    recurrence_interval=60,
    period_start="09:00",
    period_end="12:00"
    # period_days 不填表示每天
)
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `action` | str | 是 | 操作类型 | "create" |
| `title` | str | 是 | 提醒标题 | "喝水" |
| `trigger_time` | str | 是 | 触发时间 | "2025年12月24日09时00分" |
| `recurrence_type` | str | 否 | 周期类型 | "daily", "interval" |
| `recurrence_interval` | int | 否 | 间隔数 | 30 (分钟) |
| `period_start` | str | 否 | 时间段开始 | "09:00" |
| `period_end` | str | 否 | 时间段结束 | "18:00" |
| `period_days` | str | 否 | 生效日期 | "1,2,3,4,5" |

**重要说明：**
- `period_start` 和 `period_end` 必须同时设置或同时留空
- 如果只设置其中一个，系统会忽略时间段参数，创建普通提醒
- `period_days` 可以留空，表示每天生效

### period_days 值

- `"1,2,3,4,5"` - 工作日
- `"6,7"` - 周末
- `"1,3,5"` - 周一、三、五
- 不填 - 每天

## 时间格式规范

### 标准格式（推荐）

```
绝对时间：YYYY年MM月DD日HH时MM分
示例：2025年12月24日09时00分

相对时间：中文表达
示例：30分钟后、2小时后、明天、后天

时间段：HH:MM
示例：09:00、18:00
```

### 禁止格式

```
❌ "下午3点"（缺少日期）
❌ "15:00"（缺少日期）
❌ "2025-12-24 15:00"（格式错误）
❌ "12月24日15时"（缺少年份）
```

## 触发逻辑

### 普通提醒

1. 到达触发时间
2. 发送提醒消息
3. 如果是周期提醒，计算下次触发时间
4. 如果是单次提醒，标记为完成

### 时间段提醒

1. 到达触发时间
2. **检查是否在时间段内**
   - 在时间段内：发送提醒，计算下次触发时间
   - 不在时间段内：跳过，重新计算到下一个有效时间段
3. **检查是否是有效日期**（如果设置了 period_days）
   - 是有效日期：正常处理
   - 不是有效日期：跳到下一个有效日期

## 常见问题

### Q1: 如何理解"工作时间"？

A: 默认理解为：
- 时间段：09:00-18:00
- 生效日期：周一到周五

### Q2: 时间段提醒会在周末触发吗？

A: 取决于 `period_days` 设置：
- 如果设置为 "1,2,3,4,5"（工作日），周末不会触发
- 如果不设置，每天都会触发

### Q3: 如果当前不在时间段内会怎样？

A: 系统会自动跳到下一个有效时间段的开始时间，不会触发提醒.

### Q4: LLM 如何知道当前时间？

A: 系统会在 Prompt 中动态注入当前时间，LLM 根据这个时间进行推理.

### Q5: 如果不设置时间段参数会怎样？

A: 不设置 `period_start` 和 `period_end`（或都为 None）时，会创建普通提醒，不受时间段限制.

### Q6: 如果只设置了 period_start 或 period_end 其中一个会怎样？

A: 系统会忽略不完整的时间段参数，创建普通提醒，并记录警告日志.提醒仍然会被成功创建.

### Q7: period_days 可以留空吗？

A: 可以.不设置 `period_days` 表示时间段每天生效（包括周末）.

## 架构说明

### 核心组件

1. **ReminderDetectAgent** - 识别提醒意图
2. **reminder_tool** - 执行提醒操作
3. **handle_pending_reminders** - 触发提醒
4. **时间工具函数** - 时间段判断和计算

### 工作流程

```
用户输入
  ↓
ReminderDetectAgent (LLM)
  ↓ (识别意图 + 提取参数)
reminder_tool
  ↓ (创建提醒)
MongoDB
  ↓
定时任务
  ↓
handle_pending_reminders
  ↓ (检查时间段)
触发提醒 / 重新计算
```
