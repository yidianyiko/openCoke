# Tasks

This directory holds task-local work state for active and archived work items.

Use a task file when work needs a stable home for:

- goal
- scope
- acceptance criteria
- current status
- handoff notes

Execution plans belong in [`../docs/exec-plans/`](../docs/exec-plans/README.md).
Task files hold the day-to-day state of one work item.

## Relationship To `docs/superpowers/`

Historical implementation detail may still live in `docs/superpowers/` for
older work. New task-local state should live here.

Use [`_template.md`](./_template.md) for new tasks.
